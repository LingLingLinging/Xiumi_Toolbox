# 🚀 新功能使用指南

## 📋 功能概览

现在工具支持多种连接浏览器的方式，让你可以更灵活地使用：

### 🔧 四种连接方式

1. **启动新实例** - 程序自动启动新的浏览器
2. **连接已运行实例** - 连接到你已经打开的浏览器
3. **自动启动调试模式** - 程序帮你启动调试模式浏览器
4. **指定浏览器路径** - 使用自定义的浏览器路径

## 🎯 推荐使用方式

### 方式A：简单快速（适合新手）
```bash
python fetch_quickshare.py
```
选择 "3. 自动启动调试模式浏览器"

### 方式B：精确控制（适合熟练用户）
```bash
# 第一步：启动调试浏览器
python start_browser.py

# 第二步：在浏览器中登录秀米，准备好页面

# 第三步：运行主程序
python fetch_quickshare.py
# 选择 "2. 连接到已运行的浏览器"
```

## 🔍 详细说明

### 连接已运行实例的优势
- ✅ 可以复用已登录状态
- ✅ 可以手动预先导航到需要的页面
- ✅ 避免重复登录
- ✅ 可以在自己熟悉的浏览器环境中操作

### 自动启动调试模式的优势
- ✅ 一键启动，无需手动输入命令
- ✅ 自动配置调试端口
- ✅ 适合快速测试

### 指定浏览器路径的优势
- ✅ 支持便携版浏览器
- ✅ 支持自定义安装位置
- ✅ 支持企业版或特殊版本

## 📝 使用技巧

### 1. 预登录技巧
```bash
# 启动调试浏览器
python start_browser.py

# 在打开的浏览器中：
# 1. 访问 https://xiumi.us
# 2. 完成登录
# 3. 导航到文章列表页面
# 4. 然后运行主脚本
python fetch_quickshare.py  # 选择连接模式
```

### 2. 批处理技巧
如果需要多次运行，可以保持调试浏览器开启：
- 第一次运行后不要关闭浏览器
- 后续运行直接选择"连接到已运行实例"
- 节省登录时间

### 3. 故障排除
如果连接失败：
1. 确保浏览器是以调试模式启动的
2. 检查端口9222是否被占用
3. 尝试重启浏览器或使用不同端口

## ⚙️ 高级配置

### 自定义调试端口
默认使用端口9222，如有冲突可以修改：
```bash
# 启动时指定端口
python start_browser.py  # 程序会询问端口号
```

### 手动启动命令
如果你喜欢手动控制，可以直接运行：
```bash
# Edge
"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --remote-debugging-port=9222

# Chrome  
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222
```

## 🔒 安全提示
- 调试模式会开放浏览器控制接口
- 仅在本地使用，不要在公共网络环境中使用
- 使用完毕后建议关闭调试模式浏览器

## 📞 问题反馈
如果遇到问题：
1. 检查浏览器是否正确安装
2. 确认网络连接正常
3. 查看控制台错误信息
4. 尝试使用不同的连接方式
